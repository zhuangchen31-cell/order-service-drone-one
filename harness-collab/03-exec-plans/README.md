# 执行计划目录（03-exec-plans）

本目录存放所有功能的测试执行记录和发布检查清单。执行计划是 Dev_Lifecycle 测试验证阶段的产出物，也是进入文档同步阶段的前置条件。

---

## 记录规范

### 何时创建执行计划

- 开始测试验证阶段时，**必须**创建对应的执行计划文档
- 每次执行测试命令后，将结果记录到执行计划中
- 发现问题时，在问题记录表中登记，并跟踪解决状态
- 所有问题解决后，完成发布检查清单，方可进入文档同步阶段

### 文档结构要求

每份执行计划必须包含以下章节：

| 章节 | 是否必填 | 说明 |
|------|----------|------|
| 测试执行记录 | ✅ 必填 | 每次测试执行的时间、命令、结果和覆盖率 |
| 问题记录 | ✅ 必填 | 测试过程中发现的问题及解决方案 |
| 发布检查清单 | ✅ 必填 | 发布前必须完成的所有检查项 |

### 测试执行记录要求

- 每次执行测试命令后，必须记录执行时间、命令、结果和覆盖率
- 覆盖率必须达到 80% 以上才能通过 GATE-05 检查点
- 如果测试失败，记录失败原因并在问题记录中登记

### 问题记录要求

- 所有测试过程中发现的问题必须登记
- 问题严重程度分为：P0（阻断发布）、P1（高优先级）、P2（中优先级）、P3（低优先级）
- P0 和 P1 问题必须在发布前解决
- 每个问题必须记录解决方案

### 文档命名规范

- 格式：`{功能英文名}-exec-plan.md`
- 使用小写字母和连字符（kebab-case）
- 示例：`user-management-exec-plan.md`

## 与 Dev_Lifecycle 的关系

```
[编码实现阶段] → 代码和测试类已创建
      ↓
[测试验证阶段] ← 本目录文档在此阶段创建
      ↓
  GATE-05 检查点：覆盖率是否 ≥ 80%？
      ↓ 通过
[文档同步阶段] → harness-collab/04-api-docs/
```

执行计划记录了测试验证阶段的完整过程，是质量保证的重要依据。

## 常用测试命令

```bash
# 执行全量测试（harness-new 模式，覆盖率 ≥ 80% 才通过）
mvn clean verify -Pharness-new

# 执行全量测试（harness-legacy 模式，仅生成报告）
mvn clean verify -Pharness-legacy

# 仅执行单元测试（不检查覆盖率）
mvn test

# 查看覆盖率报告
open target/site/jacoco/index.html

# 执行 Checkstyle 检查
mvn checkstyle:check -Pharness-new

# 执行 SpotBugs 检查
mvn spotbugs:check -Pharness-new
```

## 模板使用

使用 `templates/exec-plan-template.md` 作为起点创建新的执行计划：

```bash
# 复制模板到当前目录
cp templates/exec-plan-template.md {功能英文名}-exec-plan.md
```

然后在测试执行过程中逐步填写各章节内容。

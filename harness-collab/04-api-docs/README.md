# API 文档目录（04-api-docs）

本目录存放所有模块的 REST API 接口文档。API 文档必须与代码保持同步，是 Dev_Lifecycle 文档同步阶段的核心产出物。

---

## 维护规范

### 何时更新 API 文档

以下情况**必须**同步更新 API 文档：

| 触发事件 | 更新内容 | 触发 Hook |
|----------|----------|----------|
| 新增 API 端点 | 在对应模块文档中新增接口描述 | `api-doc-sync-check.md` |
| 修改 API 端点（路径/参数/响应） | 更新对应接口的描述 | `api-doc-sync-check.md` |
| 删除 API 端点 | 在文档中标记为已废弃或删除 | `api-doc-sync-check.md` |
| 修改请求/响应格式 | 更新对应接口的示例 | `api-doc-sync-check.md` |

Kiro 的 `api-doc-sync-check` Hook 会在 controller 层文件被修改时自动提示开发者检查并更新 API 文档。

### 文档结构要求

每份 API 文档必须包含以下内容：

| 章节 | 是否必填 | 说明 |
|------|----------|------|
| 概述 | ✅ 必填 | 基础路径、版本、认证方式 |
| 接口列表 | ✅ 必填 | 所有端点的简要列表 |
| 接口详情 | ✅ 必填 | 每个端点的完整描述 |
| 错误码说明 | ✅ 必填 | 通用错误码和模块特定错误码 |
| 变更记录 | ✅ 必填 | API 版本变更历史 |

### 接口描述要求

每个接口必须包含：
- HTTP 方法和路径
- 功能描述
- 请求参数（路径参数、查询参数、请求头）
- 请求体（如有）及字段说明
- 响应示例（成功和失败）
- 错误码列表

### 文档命名规范

- 格式：`{模块英文名}-api.md`
- 使用小写字母和连字符（kebab-case）
- 示例：`user-api.md`、`order-api.md`、`product-api.md`

## 与代码的同步要求

### 同步检查流程

```
修改 controller 层代码
      ↓
Hook 触发：api-doc-sync-check
      ↓
检查是否有新增/修改的公共 API 端点
      ↓ 有变更
更新 04-api-docs/{模块}-api.md
      ↓
更新 func.md 中的 API 文档链接和最后更新时间
      ↓
GATE-06 检查点通过
```

### 版本兼容性要求

- API 文档中必须标注每个接口的引入版本
- 废弃的接口必须标注废弃版本和替代接口
- 破坏性变更（Breaking Change）必须在文档中明确标注

## 与 Dev_Lifecycle 的关系

```
[测试验证阶段] → 测试通过，覆盖率达标
      ↓
[文档同步阶段] ← 本目录文档在此阶段更新
      ↓
  GATE-06 检查点：API 文档是否已更新？
      ↓ 通过
  GATE-07 检查点：func.md 是否已更新？
      ↓ 通过
[CI 发布阶段]
```

## 通用响应格式

所有 API 接口使用统一的响应格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 业务状态码（非 HTTP 状态码） |
| message | String | 响应消息 |
| data | Object/Array/null | 响应数据 |

### 通用错误码

| HTTP 状态码 | 业务码 | 说明 |
|------------|--------|------|
| 200 | 200 | 请求成功 |
| 201 | 201 | 创建成功 |
| 400 | 400 | 请求参数错误 |
| 401 | 401 | 未认证（Token 无效或过期） |
| 403 | 403 | 无权限访问 |
| 404 | 404 | 资源不存在 |
| 409 | 409 | 资源冲突（如重复创建） |
| 500 | 500 | 服务器内部错误 |

## 模板使用

使用 `templates/api-doc-template.md` 作为起点创建新的 API 文档：

```bash
# 复制模板到当前目录
cp templates/api-doc-template.md {模块英文名}-api.md
```

然后按照模板中的提示填写各章节内容。

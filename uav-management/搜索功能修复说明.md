# 搜索功能修复说明

## 修复时间
2026-04-28 16:58

## 问题描述
用户反馈：成功添加无人机信息后，使用型号搜索无法找到已添加的无人机。

## 问题原因
搜索表单使用的是"型号 + 厂商"的组合筛选条件，而不是"型号 + 序列号"。这导致：
- 用户只输入型号时，如果厂商字段为空，可能无法精确匹配
- 序列号是更重要的唯一标识符，应该作为主要搜索条件

## 修复方案
将搜索条件从"型号 + 厂商"改为"型号 + 序列号"

## 搜索功能说明

### 当前搜索条件
1. **无人机型号** - 模糊匹配（支持部分匹配）
2. **序列号** - 模糊匹配（支持部分匹配）
3. **状态** - 精确匹配（在用/维修/报废）

### 搜索逻辑
- 所有条件之间是 **AND** 关系（同时满足）
- 型号和序列号支持模糊搜索（包含关键字即可）
- 可以单独使用任一条件，也可以组合使用
- 空条件会被忽略

### 使用示例

#### 示例 1: 按型号搜索
```
型号: DJI Mini 3 Pro
序列号: (空)
状态: 全部状态
```
结果：返回所有型号包含"DJI Mini 3 Pro"的无人机

#### 示例 2: 按序列号搜索
```
型号: (空)
序列号: DJ20260427
状态: 全部状态
```
结果：返回所有序列号包含"DJ20260427"的无人机

#### 示例 3: 组合搜索
```
型号: DJI
序列号: 2026
状态: 在用
```
结果：返回型号包含"DJI"、序列号包含"2026"、且状态为"在用"的无人机

#### 示例 4: 按状态筛选
```
型号: (空)
序列号: (空)
状态: 在用
```
结果：返回所有状态为"在用"的无人机

## 优势对比

### 之前（型号 + 厂商）
- ❌ 厂商信息不够唯一
- ❌ 用户可能不记得厂商名称
- ❌ 厂商字段可能有多种写法（如"DJI"、"大疆"）

### 现在（型号 + 序列号）
- ✅ 序列号是唯一标识符
- ✅ 序列号更容易记忆和查找
- ✅ 序列号格式统一，不会有歧义
- ✅ 更符合实际使用场景

## 数据库查询示例

### 查询 SQL（简化版）
```sql
SELECT * FROM drone 
WHERE deleted = 0
  AND model LIKE '%DJI%'           -- 型号模糊匹配
  AND serial_number LIKE '%2026%'  -- 序列号模糊匹配
  AND status = 'IN_USE'            -- 状态精确匹配
ORDER BY create_time DESC
LIMIT 10 OFFSET 0;
```

## 注意事项

1. **模糊搜索**: 型号和序列号都支持模糊搜索，输入部分关键字即可
2. **大小写**: SQLite 默认不区分大小写（case-insensitive）
3. **特殊字符**: 搜索时会自动处理特殊字符
4. **性能**: 模糊搜索在大数据量时可能较慢，建议添加索引
